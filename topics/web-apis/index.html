External Web APIs

With many Web-Apps we want to reach farther than our locally hosted services and reach APIs hosted by other platforms and services outside our network. While there are a number of them, many are just first version and others (like Facebook) are still absent, leaving a lot of room for improvement.

Connecting to the outside world is important, too many modern web applications are useless without an external party they connect to and interact with. This is an incomplete list of API(-wrapper)-Implementations in Rust:

  • acme-client

    Easy to use ACME client library to issue, renew and revoke TLS certificates

  • aspotify

    Asynchronous client for the Spotify API

  • twitter-api

    Unofficial Rust library for the Twitter API.

  • discord

    Client library for the Discord API

  • strava

    Strava API v3 Client

  • hubcaps

    Rust interface for Github

  • googl

    A simple interface to the Google URL Shortener API.

  • vkrs

    Vkontakte API Rust client library

  • telegram-bot

    A library for creating Telegram bots

  • urlshortener

    A very simple url shortener client library

  • wikipedia

    Access wikipedia articles from Rust

  • keen

    keen.io api v3.0 for rust

  • pusher

    The Rust library for interacting with the Pusher HTTP API.

  • slack-hook

    A rust crate for sending messages to Slack via webhooks.

  • rants

    An async NATS client library.

  • rusoto_core

    AWS SDK for Rust - Core
